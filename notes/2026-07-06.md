## 🔴 Critical Issues

### 1. Zero ErrorBoundaries
`ErrorBoundary` is completely absent. Any runtime error in any node or component → white screen.
**Fix:** Wrap CanvasPage, QuizInteraction, and the root with error boundaries showing a recovery UI.

### 2. Pipeline Has No Checkpointing
`runPipeline` in `pipeline.ts` does: detail → quiz → summary → build graph. This is 5+ sequential LLM calls. If call #2 fails, the entire session is lost — the first call's output is discarded.
**Fix:** Persist intermediate results to IDB per-step. Retry failed steps individually with exponential backoff.

### 3. All Four Parsers Are Fragile and Duplicated
`extractBalanced` is copy-pasted identically in 4 files (`outlineParser.ts`, `contentParser.ts`, `summaryParser.ts`, `gradeParser.ts`). None use schema validation (Zod/yup). LLMs occasionally output malformed JSON — current regex/brute-force extraction silently fails or produces `undefined`.
**Fix:** Extract shared utility, add Zod schemas for every response shape, enforce `response_format: { type: 'json_object' }` on all LLM calls.

### 4. Test Coverage Near Zero
2 test files for ~45 production files. The most critical code (parsers, pipeline, stores, `useQuizAnswer`) has zero tests.
**Fix:** Start with parser unit tests (pure functions), then store tests with `fake-indexeddb`, then component tests for quiz formats.

### 5. God Component in App.tsx
`App.tsx` handles: theme, page routing, URL submission, fetch orchestration, pipeline progress, session management, canvas restoration, error display. ~330 lines with 6 `useState`/`useEffect`/`useRef` hooks and a single `handleGenerate` that chains fetch → outline → pipeline.
**Fix:** Extract `useGenerateSession` hook, extract `PageRouter` component, extract `ProgressTracker`.

## 🟡 Medium Issues

### 6. Sequential Pipeline vs Parallelism
Pipeline does concept detail expansion one-by-one (sequential Promises). For 6 concepts, this is 6 sequential LLM calls.
**Fix:** Batch concepts into groups of 2-3 and fire parallel LLM calls, then merge results.

### 7. Quiz Format Components Use Inline Styles
MultipleChoice, TrueFalse, FillBlank, ShortAnswer, FreeText, Ordering all use `style={{}}` instead of CSS modules. Breaks CSS variable consistency and theme switching.
**Fix:** Convert to CSS modules matching the rest of the app.

### 8. No Loading / Error States in Quiz
`QuizInteraction` and `SummaryQuizInteraction` have no skeleton loading states during submission. The `submitting` boolean exists but isn't surfaced visually for all formats.

### 9. Dynamic Import in useQuizAnswer.ts
```ts
const settings = (await import('@/shared/stores/settingsStore')).useSettingsStore.getState();
```
Should be a static import at the top of the file.

### 10. SummaryNode Has Dead Handler
`onClick={() => {}}` on the "Take Final Quiz" button — no-op placeholder.

### 11. Undefined CSS Variables
`--dur-fast`, `--ease-out`, `--dur` are used throughout CSS modules but are **not defined** in `tokens.css`. These cascade from browser defaults, which means animations/timings may not be intentional.

## 🔵 Strengths Worth Protecting

- **Clean type system** — `NodeData` discriminated union with `kind` field is correct and safe
- **Provider abstraction** — `providers.ts` with `ProviderConfig` makes adding new LLMs trivial
- **Fetch chain with fallbacks** — Jina → multiple CORS proxies → Cloudflare proxy → LLM extraction is robust
- **CSS token system** — `tokens.css` with semantic variables (not raw colors) ensures theming consistency
- **Session persistence** — Layered approach (sessionStorage for tab resume, IDB for session history, localStorage for settings) is solid
- **Zustand stores** — Clean separation, no unnecessary re-renders, proper async loading patterns

## App Flow

```
App.handleGenerate(url):
  fetchSourceContent(url)                   → stage: 'fetch'
  chat(outlinePrompt) → parseOutline()      → stage: 'outline'
  createSession({nodes, edges})              → sessionStore.create
  select(session.id)                          → sets currentId
  setPage('canvas')                            → shows canvas immediately
  FOR EACH concept:
    chat(contentPrompt) → parseContentResponse() → stage: 'concept'
    updateCurrent(sessionId, patch)              → incremental canvas update
    sleep(2000)                                    → rate-limit delay
  chat(summaryPrompt) → parseSummaryResponse() → stage: 'summary'
  updateCurrent(sessionId, finalPatch)
  select(session.id)                              → re-pin store
  catch(error) → setPage('welcome') + setError
```

## Fetch Chain (`src/lib/fetchSourceContent.ts`)

1. IndexedDB cache check (24hr expiry) → return if fresh
2. Vite dev proxy (`/__proxy?url=`) in dev mode
3. Jina AI reader (`r.jina.ai`) with optional Bearer token
4. 8 CORS fallback proxies in sequence (allorigins, corsproxy, cors-eu, codetabs, corslol, corsfix, cfproxy)
5. LLM fallback — ask selected provider to summarize URL
6. Cache result asynchronously to IDB, truncate via `truncateByParagraphs`
